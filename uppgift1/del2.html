1- Förklara kort vad ett element, attribut och värde är för något.
Svar/ I en webbsida kan vi ha många HTML-element som skriva inom två taggar, på följande sätt, <element>. HTML-elementen har olika atribut och varje atribut kan ha olika värden.
Till exempel, HTML-elementet <img>, kan ha atributet src, (source) där vi kan ha själva länken till bilden.
Länken till bilden, blir då ett värde till src.
Här finns ett HTML-element med ett värde. <img src="iamavalutothesource.png">.

Svar/ HTML är en protokol, den står för Hyper Text Transfer Protocol.
Vi använder den för att skicka (request) via webbläsare till en IP-address eller till en domän för att kunna få tillgäng till webbsidor som finns på olika webbserver.


3- Vilka är W3C och vilken är deras huvudsakliga uppgift?
Svar/ World Wide Web Consortium
W3C finns för att kunna göra WWW internationelt standard, för till exempel html och css, vilket är bra, för att stora företag som till exempel Microsoft som skapar olika webbläsare, de kommer att följa det som är standard.
Chrome, FireFox och Edge, alla de tre nämnda webbläsarna användar det som är standard i sina renderingsmotorer i webbläsarna.
Till exempel, CSS3 har nu blivit standard accepterat, W3C har implementeras CSS3 i sina webbvaliderings applikationer.


4- Vad står WWW för och vad är detta för något?
Svar/ World Wide Web
Vi kan använda WWW tillsammans med HTTP protokolet för att kunna skicka förfrågan till olika webbsidor.
Det kan till exempel vara olika html document som har publicerats på olika publika webbserver.

7- Vilken modell bygger HTTP på?
Modellen beskriver förhållandet mellan en dator som svarar och en dator som ställer förfrågan om att få se webbplatserna.
Svar/ Jag ska försöka förklara det så simpelt det går, hoppas det blir tydlig begriplig.
Vi har våra datorer anslutan till internet, vi får en publik IP-adress från internet leverantören,
Via den pubika IP addresen, vi kommunicera med andra datorer, webserver samt vi kan ha tillgång till webbsidor. Normalt använder vi olika webbläsare för att kunna ha tillgång till webbsidor.
Vi kan använda http prokollet för att kunna begära data från med en webbserver, en centraliserad webbserver som har gjort sina webbsidorna publika via dera publika IP.
För att IP är svåra, vi kan ha domäner som pekar på IP adressen.
Datan som används som en webbservern har standard port 8080 öppen för http och 443 för https.
Till exempel, myipfs.se är ett domännamn som pekar på en IP adress.
När vi skickar en http request till domännamnet, http://myipfs.se, en request skickas till webbservern.
Webbservern svarar med att skicka innehållet, webbsidorna som finns publicerat till den som skickat förfrågan.
